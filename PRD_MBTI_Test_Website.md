**문서 버전**: 1.0
**작성일**: 2025-12-11
**작성자**: Senior Web Service Planner

---

## 1. 프로젝트 개요

### 1.1 서비스명

**MBTI Lab** (가칭)

### 1.2 한 줄 소개

누구나 3분 안에 간편하게 자신의 MBTI 성격 유형을 확인하고 결과를 공유할 수 있는 웹 기반 심리 테스트 서비스

### 1.3 서비스 비전 및 미션

**비전**
"자기 이해의 첫 걸음, 누구나 쉽게 접근할 수 있는 MBTI 테스트 플랫폼"

**미션**

- 복잡한 회원가입 없이 즉시 테스트 가능한 접근성 제공
- 과학적 근거에 기반한 정확한 성격 유형 판별
- 직관적이고 재미있는 사용자 경험 제공
- 결과 공유를 통한 자연스러운 바이럴 확산

### 1.4 핵심 가치 제안 (Value Proposition)

| 사용자 Pain Point                                      | 우리의 솔루션                     |
| ------------------------------------------------------ | --------------------------------- |
| 기존 MBTI 테스트는 질문이 너무 많아 지루함 (60-90문항) | 12문항으로 압축한 Quick Test      |
| 회원가입 절차가 번거로움                               | 회원가입 없이 바로 테스트 시작    |
| 결과를 친구와 공유하기 어려움                          | 이미지 저장 및 SNS 공유 버튼 제공 |
| 테스트 결과가 텍스트 위주로 지루함                     | 시각적으로 매력적인 결과 페이지   |
| 자신의 유형이 얼마나 흔한지 궁금함                     | 실시간 통계 제공                  |

### 1.5 경쟁 서비스 분석 및 차별화 포인트

**주요 경쟁 서비스**

- 16Personalities (영문, 60문항)
- 국내 각종 MBTI 테스트 사이트 (질문 수 과다, UI 복잡)

**차별화 포인트**

1. **최소 질문 수**: 12문항으로 3분 내 완료
2. **제로 프릭션**: 회원가입, 개인정보 수집 없음
3. **공유 최적화**: 인스타그램 스토리, 카카오톡 공유에 최적화된 이미지 생성
4. **모바일 퍼스트**: 모바일 환경에서의 완벽한 UX
5. **실시간 통계**: "당신과 같은 유형은 전체의 X%입니다" 실시간 데이터

### 1.6 예상 비즈니스 모델

**Phase 1 (MVP)**: 완전 무료, 수익 모델 없음

- 목표: 사용자 확보 및 트래픽 증대
- MAU 10,000 이상 달성 시 Phase 2 고려

**Phase 2 (향후)**: 선택적 수익화

- Google AdSense 광고 (비침해적 위치)
- 프리미엄 상세 리포트 (유료)
- B2B 기업용 팀 분석 서비스

## 2. 타겟 사용자 분석

### 2.1 주요 타겟 페르소나

#### 페르소나 1: 호기심 많은 대학생 지민 (22세)

**인구통계**

- 나이: 22세, 여성
- 직업: 대학생 (심리학과 3학년)
- 디바이스: 주로 모바일 (iPhone)

**특성**

- SNS 활동 활발 (인스타그램 하루 2-3회 포스팅)
- 심리 테스트, 성격 분석에 관심 많음
- 친구들과 결과 공유하며 대화하는 것을 즐김

**Pain Points**

- 긴 테스트는 집중력이 떨어져 중도 포기
- 결과를 예쁘게 꾸며서 공유하고 싶음
- 친구들이 어떤 유형인지 비교하고 싶음

**서비스 사용 동기**

- 자신의 성격 유형 확인 및 자기 이해
- 인스타그램 스토리에 올릴 콘텐츠 확보
- 친구들과 MBTI 이야기하며 친밀감 형성

**사용 시나리오**

1. 인스타그램에서 친구가 공유한 링크 클릭
2. 3분 안에 테스트 완료
3. 결과 이미지 저장 후 인스타그램 스토리에 업로드
4. 친구들에게도 링크 공유

### 2.3 TAM/SAM/SOM 추정

**TAM (Total Addressable Market)**

- 한국 인터넷 사용자: 약 5,000만 명
- MBTI에 관심 있는 인구: 추정 60% = 3,000만 명

**SAM (Serviceable Addressable Market)**

- 온라인 심리 테스트 이용 경험자: 추정 30% = 1,500만 명
- 연간 최소 1회 MBTI 테스트 검색/시도 인구: 약 800만 명

**SOM (Serviceable Obtainable Market)**

- 1차년도 목표 사용자 수: 10만 명 (SAM의 1.25%)
- 월간 활성 사용자 (MAU): 5만 명
- 일간 활성 사용자 (DAU): 5,000명

## 3. 핵심 기능 목록

### 3.1 기능 우선순위 분류

#### P0 (MVP 필수) - 서비스 런칭에 반드시 필요

##### 3.1.1 테스트 진행 기능

**기능명**: 12문항 MBTI 질문 및 응답 시스템

**상세 설명**

- 4개 차원(E/I, S/N, T/F, J/P) × 3문항 = 총 12문항
- 각 질문은 5점 척도 (매우 아니다 / 아니다 / 보통 / 그렇다 / 매우 그렇다)
- 질문 순서는 차원별로 섞여서 제시 (E→S→T→J→I→N→F→P→E... 패턴)
- 뒤로가기 버튼으로 이전 질문 수정 가능
- 응답하지 않은 질문이 있으면 다음으로 진행 불가

**사용자 스토리**

> As a 사용자, I want 각 질문에 5점 척도로 답변할 수 있어야 so that 나의 성격을 정확하게 표현할 수 있다.

**수용 기준 (Acceptance Criteria)**

- [ ] 12개 질문이 순차적으로 표시됨
- [ ] 각 질문마다 5개 선택지 버튼 제공
- [ ] 선택 시 버튼 색상 변경으로 선택 상태 표시
- [ ] 이전 질문으로 돌아가기 가능
- [ ] 모든 질문 응답 완료 시에만 결과 보기 버튼 활성화

---

##### 3.1.2 프로그레스 바

**기능명**: 테스트 진행률 시각화

**상세 설명**

- 화면 상단에 고정된 프로그레스 바 (진행률 % 표시)
- 현재 질문 번호 / 전체 질문 수 표시 (예: "3/12")
- 진행률에 따라 프로그레스 바 색상 그라데이션 적용

**사용자 스토리**

> As a 사용자, I want 테스트가 얼마나 남았는지 확인할 수 있어야 so that 중도 이탈 없이 끝까지 완료할 수 있다.

**수용 기준**

- [ ] 질문 넘길 때마다 프로그레스 바 업데이트
- [ ] 진행률 퍼센트 정확하게 표시 (예: 25%, 50%, 75%)
- [ ] 모바일에서도 프로그레스 바 잘 보임
- [ ] 뒤로가기 시 프로그레스 바도 감소

---

##### 3.1.3 결과 계산 로직

**기능명**: MBTI 유형 판별 알고리즘

**상세 설명**

- 각 차원별 3문항의 점수를 합산
- E/I: E 성향 질문 점수 vs I 성향 질문 점수 비교
- S/N, T/F, J/P도 동일한 방식
- 점수가 동점일 경우 해당 차원은 중립(N)으로 처리하되, 최종 결과에서는 약간 높은 쪽으로 배정
- 결과는 16가지 유형 중 하나 (INTJ, ENFP 등)

**사용자 스토리**

> As a 시스템, I want 사용자의 응답을 분석하여 정확한 MBTI 유형을 도출해야 so that 신뢰할 수 있는 결과를 제공할 수 있다.

**수용 기준**

- [ ] 12문항 응답 기반으로 4개 차원 점수 계산
- [ ] 각 차원별 우세 성향 판별 (예: E > I이면 E 선택)
- [ ] 동점 처리 로직 구현 (기본값: N, F, P 우선)
- [ ] 최종 4글자 MBTI 코드 생성 (예: ENFP)

---

##### 3.1.4 결과 페이지

**기능명**: MBTI 유형별 상세 정보 표시

**상세 설명**
각 유형별로 다음 정보 제공:

- **유형명**: "ENFP - 재기발랄한 활동가" (별칭)
- **한 줄 요약**: 핵심 특징 30자 이내
- **성격 특징**: 3-5개 키워드 (예: 열정적, 창의적, 사교적)
- **강점**: 3-4개 항목 (불릿 포인트)
- **약점**: 3-4개 항목 (불릿 포인트)
- **추천 직업**: 5-7개 직업군
- **유명인**: 해당 유형으로 알려진 유명인 3-5명
- **궁합**: 최고 궁합 2개 유형, 최악 궁합 1개 유형

**사용자 스토리**

> As a 사용자, I want 내 MBTI 유형의 상세한 설명을 확인하고 싶어 so that 나 자신을 더 깊이 이해할 수 있다.

**수용 기준**

- [ ] 16개 유형 모두 상세 정보 데이터 준비
- [ ] 결과 페이지에 유형 정보 깔끔하게 레이아웃
- [ ] 모바일 스크롤로 모든 정보 확인 가능
- [ ] 정보 출처 명시 (하단 작은 글씨)

---

##### 3.1.5 결과 이미지 생성

**기능명**: 공유용 이미지 자동 생성

**상세 설명**

- Canvas API를 사용하여 클라이언트 측에서 이미지 생성
- 이미지 크기: 1080x1920px (인스타그램 스토리 비율)
- 포함 요소:
  - MBTI 유형 코드 (큰 글씨)
  - 유형 별칭 (예: "재기발랄한 활동가")
  - 3가지 대표 키워드
  - 서비스 로고 및 URL (하단)
  - 배경: 유형별 고유 색상/패턴
- 다운로드 형식: PNG

**사용자 스토리**

> As a 사용자, I want 내 결과를 예쁜 이미지로 저장하고 싶어 so that SNS에 공유할 수 있다.

**수용 기준**

- [ ] "이미지 저장" 버튼 클릭 시 이미지 다운로드
- [ ] 이미지에 유형 정보 및 로고 표시
- [ ] 이미지 해상도 1080x1920px (모바일 최적화)
- [ ] PNG 형식으로 다운로드
- [ ] 16개 유형별 고유한 디자인 템플릿

---

##### 3.1.6 통계 기능

**기능명**: 유형별 분포 통계 실시간 표시

**상세 설명**

- 지금까지 테스트를 완료한 사람들의 유형 분포 표시
- 막대 그래프 또는 파이 차트로 시각화
- 사용자의 유형이 전체에서 차지하는 비율 강조 표시
- 예: "ENFP는 전체의 12.3%입니다"
- 통계는 익명으로 수집 (IP 기반 중복 방지, 개인정보 미수집)

**사용자 스토리**

> As a 사용자, I want 내 유형이 얼마나 흔한지 궁금해 so that 다른 사람들과 비교해볼 수 있다.

**수용 기준**

- [ ] 백엔드에서 유형별 카운트 저장
- [ ] 결과 페이지에 막대 그래프로 16개 유형 분포 표시
- [ ] 사용자 유형 막대는 하이라이트 처리
- [ ] 퍼센트 표시 (소수점 1자리)
- [ ] 테스트 완료 시 자동으로 통계 업데이트

#### P1 (중요) - 사용자 경험 향상에 중요

##### 3.1.7 SNS 공유 버튼

**기능명**: 카카오톡 및 인스타그램 공유 기능

**상세 설명**

**카카오톡 공유**

- Kakao SDK를 사용한 카카오톡 공유
- 공유 내용:
  - 제목: "나는 [MBTI 유형] - [별칭]"
  - 설명: "당신의 MBTI는 무엇인가요?"
  - 썸네일 이미지: 유형별 대표 이미지
  - 버튼: "나도 테스트하기" (링크)

**인스타그램 공유**

- 모바일 환경: 이미지 다운로드 후 인스타그램 앱 자동 실행
- 데스크톱 환경: "이미지를 저장한 후 인스타그램에 업로드하세요" 안내 메시지

**사용자 스토리**

> As a 사용자, I want 내 결과를 친구들에게 쉽게 공유하고 싶어 so that 친구들도 테스트를 해보도록 유도할 수 있다.

**수용 기준**

- [ ] 카카오톡 공유 버튼 클릭 시 카카오톡 앱/웹 실행
- [ ] 카카오톡 메시지에 테스트 링크 포함
- [ ] 인스타그램 공유 버튼 클릭 시 이미지 다운로드 + 앱 실행
- [ ] 데스크톱에서는 적절한 안내 메시지 표시
- [ ] 공유 시 UTM 파라미터로 유입 경로 트래킹

#### 5.3.1 브랜드 톤앤매너

**브랜드 성격**

- 친근함: 심리 테스트를 무겁지 않게, 친구와 대화하는 느낌
- 신뢰성: 과학적 근거를 바탕으로 한 정확성 강조
- 현대적: 트렌디한 디자인, 2030 세대 감각
- 명쾌함: 복잡하지 않고 직관적인 UI

**카피 톤**

- 반말 사용 (친근함)
- 명확하고 간결한 문장
- 긍정적이고 격려하는 어조
- 예시:
  - Good: "3분 만에 확인하세요!"

1. **색상 대비**

   - 본문 텍스트: 최소 4.5:1 대비
   - 큰 텍스트(18px+): 최소 3:1 대비
   - 버튼/UI 요소: 최소 3:1 대비

2. **키보드 접근성**

   - 모든 인터랙티브 요소는 Tab 키로 접근 가능
   - Focus 상태 명확한 시각적 표시 (Outline)
   - Skip to content 링크 제공

3. **스크린 리더 지원**

   - 모든 이미지에 alt 텍스트
   - ARIA 레이블 사용 (aria-label, aria-describedby)
   - 랜드마크 역할 명시 (role="main", role="navigation")

4. **반응형**

   - 텍스트 200% 확대 시에도 레이아웃 유지
   - 모바일 터치 타겟 최소 44x44px

5. **언어**
   - HTML lang 속성 명시 (lang="ko")

**접근성 체크리스트**

- [ ] 모든 버튼에 명확한 레이블
- [ ] 프로그레스 바에 aria-valuenow, aria-valuemin, aria-valuemax
- [ ] 에러 메시지는 aria-live="polite"로 스크린 리더에 즉시 전달
- [ ] 이미지는 decorative일 경우 alt="" (빈 값)
- [ ] 링크는 "여기 클릭" 대신 목적지 명확히 표시

**목적**: 테스트 결과 제출 및 통계 업데이트

**요청 헤더**

```
Content-Type: application/json
```

**요청 본문**

```json
{
  "mbtiType": "ENFP",
```

```json
{
  "success": true,
  "totalTests": 75297
}
```

**응답 (실패, 400 Bad Request)**

```json
{
  "error": "Invalid MBTI type",
  "message": "mbtiType must be one of 16 valid types"
}
```

**비즈니스 로직**

1. `mbtiType` 유효성 검사 (16개 유형 중 하나)
2. IP 해시 생성 (SHA-256)
3. 중복 체크 (동일 IP에서 1시간 내 재제출 방지, 선택)
4. DB에 레코드 삽입
5. 통계 캐시 무효화

**인증**: 없음

**Rate Limiting**: IP당 시간당 5회

- 평균 방문 횟수

**수익 (Revenue)** - Phase 3

- 유료 전환율
- ARPU (Average Revenue Per User)
- MRR (Monthly Recurring Revenue)
